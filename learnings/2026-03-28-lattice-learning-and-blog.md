# Session: 2026-03-28 - Lattice Learning Framework + Blog Post

## What Worked
- Interactive tutoring session (5 Big Ideas with analogies) was highly effective for the user's understanding
- Using existing mental-models insights (Munger) as bridges to AI concepts created immediate "aha" moments
- The framework spec (`specs/lattice-learning/design.md`) makes this reusable for future domains
- Copy-pasting from styled HTML into Substack preserves table formatting perfectly

## What Didn't Work
- Dramatic/binary hooks for writing ("couldn't explain what a harness is", "couldn't see how any connected") — flagged 3x by user as oversimplification
- Removing tables for X compatibility then needing to restore them for Substack — should have maintained separate versions from the start
- Pasting markdown into X article editor creates double-spaced paragraphs — need plain text version

## Key Insight
When writing about someone's learning journey, frame the problem at the level of synthesis ("struggling to put it all together") not vocabulary ("couldn't explain term X") or binary failure ("couldn't connect anything"). Real experiences are gradients. Dramatic framing attracts attention but sacrifices authenticity — and this user explicitly prioritizes authenticity.
